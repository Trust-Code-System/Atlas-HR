# Runbook: Support Inbox Overflowing

## Symptoms

- Open support tickets exceed same-day response capacity.
- Multiple users report the same issue.
- Product Hunt or LinkedIn comments point to confusion in the product.

## Immediate Actions

1. Open `/admin/tickets`.
2. Sort by urgent, billing, and technical tickets first.
3. Group duplicate issues by theme.
4. Write canned responses for top repeated issues.
5. Add a short Help Center article if the answer is reusable.

## Priority Order

1. Billing or paid-access blockers.
2. Sign-up, login, or onboarding blockers.
3. Document generation or Copilot failures.
4. Data privacy or deletion requests.
5. Feature questions and general feedback.

## Communication

- Acknowledge quickly even if resolution takes longer.
- Be specific about next steps and expected timing.
- Do not overpromise. If it needs engineering work, say so.

## Resolution

- Convert repeated tickets into help articles.
- Add UI copy where users are getting stuck.
- Link known incidents to the status page.
- Track the top five themes in the daily launch notes.
