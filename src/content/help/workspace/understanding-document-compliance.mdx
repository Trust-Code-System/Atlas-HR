Document compliance in Atlas HR helps you track which employees have the required documents in place — and which ones are missing, expiring, or overdue. Here's how it works.

## What is the compliance tab?

The **Compliance** tab (found in the People section) gives you a bird's-eye view of your workforce's document status. It shows every employee alongside their compliance status for each required document type.

You can also see an individual employee's compliance status from their profile → **Documents** tab.

## Document types tracked

Atlas HR can track any document type you configure. Common examples include:

- **Right to work / work authorisation** — typically has an expiry date for visa holders
- **Employment contract** — a signed version should be on file
- **NDA / confidentiality agreement** — signed copy required
- **Health and safety induction** — certifying training completion
- **DBS / background check** — expiry varies by role and country
- **Professional licence or certification** — for regulated roles (e.g. nurse, solicitor, accountant)
- **Emergency contact form** — no expiry, but required on file
- **Probation review sign-off** — triggered by probation end date

To configure which document types are required and for which roles, go to **Settings → Compliance → Document requirements**.

## Status indicators

Each document has a traffic-light status:

| Status | Meaning |
|---|---|
| **Green** | Document is on file and not expiring soon |
| **Amber** | Document is expiring within 30 days (or missing a soft-required document) |
| **Red** | Document has expired, or a required document is missing |

<Callout type="warning">
Red status on a right-to-work or work authorisation document is a legal risk. Atlas HR will flag it, but the responsibility for action rests with you. Don't leave red compliance statuses unresolved — contact the employee immediately.
</Callout>

## Expiry alerts

When a document is approaching its expiry date, Atlas HR can automatically:

- **Email the employee** — prompting them to upload a renewed document
- **Email the HR Admin** — so you're aware and can chase if needed
- **Create a task** — on the employee's onboarding/task list

Alert timings are configurable. Default is 30 days before expiry, with a second alert at 14 days. Change these in **Settings → Compliance → Alert settings**.

## Marking a document as reviewed

When an employee uploads a document via the portal, it lands with a "Pending review" status. An Admin needs to:

1. Open the employee profile → Documents tab.
2. Click the document marked "Pending review."
3. Review the document.
4. Click **Approve** (status becomes Green) or **Reject** with a note (the employee is notified and prompted to re-upload).

If you have an approval workflow set up for document submissions, this review step happens within the workflow rather than manually.

## Generating a compliance report

To see overall compliance across your workforce:

1. Go to **Reports → Compliance report**.
2. Filter by document type, department, or status.
3. Export as CSV if you need to share it with leadership or auditors.

<Callout type="tip">
Run the compliance report before your next audit or renewal cycle. It's much easier to chase missing documents proactively than to discover gaps under pressure.
</Callout>
