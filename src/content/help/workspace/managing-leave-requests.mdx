Atlas HR's leave management system connects employees, managers, and HR in one place. This article covers how requests flow from submission to approval.

## The leave calendar

The **Leave calendar** (People → Leave) shows all leave across your workforce in a calendar view. You can:

- See who is off on any given day
- Filter by team or department
- View leave by type (annual leave, sick leave, parental leave, etc.)
- Spot potential coverage gaps

The calendar is useful during planning seasons — particularly useful before Christmas, summer, or high-demand project periods.

## Leave types

Atlas HR supports any leave type you configure. Common types include:

- Annual leave
- Sick leave
- Parental leave (maternity, paternity, shared parental)
- Compassionate leave
- Public holidays (auto-populated based on employee's country)
- Time off in lieu (TOIL)
- Study leave

Configure leave types and their rules (accrual, maximum, carry-over) in **Settings → Leave policy**.

## How employees submit a request

Employees submit leave requests from the **employee portal** (not the main Atlas HR admin interface). See [Submitting a leave request](/help/articles/submitting-a-leave-request) for the employee-side walkthrough.

From the HR side, you'll see new requests appear in the **Leave** tab as "Pending."

## Approving or declining a request

<Steps>
1. Go to **People → Leave**.
2. Click the **Pending requests** tab.
3. Click the request to open its details.
4. Review the dates, leave type, and reason (if provided).
5. Check the calendar view to see if there are coverage conflicts.
6. Click **Approve** or **Decline**.
7. If declining, add a note explaining why — this is sent to the employee.
</Steps>

If you have an approval workflow configured, leave requests may route through a manager first before reaching HR. In that case, your HR view shows requests that have completed the workflow approval chain.

## Leave request statuses

| Status | Meaning |
|---|---|
| **Pending** | Submitted and awaiting first approver action |
| **In review** | Moving through a multi-step approval workflow |
| **Approved** | All approvers signed off — leave is confirmed |
| **Declined** | Rejected by an approver — employee is notified |
| **Cancelled** | Employee cancelled the request themselves |

## Adjusting leave balances manually

Sometimes you need to adjust an employee's leave balance — for example, when they join mid-year and should have a pro-rated entitlement, or when carry-over leave needs to be added.

1. Open the employee's profile.
2. Click the **Leave** tab.
3. Click **Adjust balance** next to the relevant leave type.
4. Enter the adjustment (positive or negative days) and a reason.
5. Save.

<Callout type="warning">
Manual balance adjustments bypass the automatic accrual calculation. Only use them for genuine corrections, and document the reason — the adjustment history is visible in the employee's leave record.
</Callout>

## Exporting leave data

For payroll runs or finance reporting, export leave data from **Reports → Leave report**. Filter by date range and leave type, then export as CSV.
